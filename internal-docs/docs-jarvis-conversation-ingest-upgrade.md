# Jarvis 真实对话自动抽取链升级说明

## 本轮新增

新增脚本：
- `jarvis_conversation_ingest.py`

## 当前能力

它会把一段真实对话文本直接做四件事：

1. 基于规则识别偏好 / 规则 / 目标
2. 结构化写入 SQLite 主表
3. 同步写入对应 history 表
4. 同步写入 intake_queue，并标记为已处理

## 价值

这意味着 Jarvis 开始从：
- 手动写偏好
- 手动跑摘要

升级为：
- 直接吃真实对话
- 自动抽取
- 自动结构化入库
- 自动生成变更历史

## 当前限制

本轮仍是规则驱动抽取，不是语义模型抽取。
但这是“真实对话 → 自动学习”闭环的第一版可运行链路。

## 下一步建议

1. 增强正则/规则集
2. 对同义表达做规范化
3. 增加冲突检测
4. 将对话 ingest 自动接到实际会话日志流
